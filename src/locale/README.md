# Locale / 国際化 (i18n)

このディレクトリでは、アプリケーションの多言語対応に関する設定および翻訳ファイルを管理します。

## 技術スタック

- **フレームワーク**: [@angular/localize](https://angular.dev/guide/i18n)
- **ファイル形式**: XLF (`.xlf`)
- **ソースロケール**: `ja` (日本語)

## 翻訳ルール

### カスタムIDの付与

新しい翻訳テキストを追加する際は、メンテナンス性を確保するため、必ず **`@@`** プレフィックスを使用して**カスタムID**を付与してください。

**例:**

```html
<h1 i18n="@@home.welcome_message">ようこそ</h1>
```

## 運用フロー

### 1. 翻訳テンプレートの更新

ソースコードやテンプレート内のテキストを変更・追加した後は、以下のコマンドを実行して `messages.xlf` を更新してください。

```bash
npm run i18n
```

### 2. 翻訳テキストの編集

各言語の翻訳作業（`.xlf` ファイルの編集）には **BabelEdit** を使用してください。BabelEdit を使用することで、複数の言語ファイルを一括で管理し、構造を維持したまま安全に編集できます。
